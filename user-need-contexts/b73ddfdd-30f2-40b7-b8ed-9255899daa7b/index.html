User Need Context: Content (Static)

There are 69 guidance statements that support this user need context:

  • Synchronized auditory alternatives - As a user with limited hearing or limited language skills, I need alternate content, such as subtitles, to be synchronized with the main content.
  • No hearing equivalents - As a user without hearing or with limited hearing, I need auditory content presented programmatically or I need equivalent visual and tactile versions.
  • Perceive auditory content - As a user with low or no hearing, I need visual or tactile equivalents for auditory content.
  • Output modality choice - As a user with limited or no vision or with limited or no hearing, I need to be able to choose modalities to be used for outputs from the system.
  • Audio characteristics - As a user with limited hearing, I need specific characteristics of audio content, cues, and feedback to fit my needs or to be adjustable.
  • Monoaural information - As a user with limited hearing, I need to have multi-channel audio information, including directional information, available in a monoaural form.
  • Multi-frequency events - As a user with limited hearing, I audio alerts and other events to be multi-frequency, including frequencies where my hearing may be better.
  • No tone differentiation - As a user who cannot differentiate quickly between different tones (frequencies), I need alerts, notifications, and other audio events to have characteristics other than tone differentiation if I need to distinguish between different audio events or obtain information from such audio events.
  • Sign language characteristics - As someone who understands sign language, I need sign language content in a sign language that I understand, with sufficient size, resolution, speed, and contrast.
  • Background noise - As a user with limited vision, hearing or language skills, I need to be able to distinguish visual, auditory, or tactile information in the foreground from background noise (noise which may be of any modality).
  • Adjustable volume - As a user with limited hearing or in an environment that makes it difficult to hear, I need to be able to adjust the volume of the system to a suitable level.
  • Select audio stream - As someone with no or limited vision or with limited hearing, I need to have the ability to select and deselect different audio streams.
  • No-color equivalents - As a user with low vision or color vision deficiency, I need information provided through chromaticity to be available in another visual manner.
  • Contrast and scale - As a user with low vision, I need text and layout design to appear in a custom or standard high contrast mode and at higher scale.
  • 2D alternative to 3D - As someone who cannot perceive content in three dimensions or who is sensitive to electronic 3D displays, I need to have three dimensional visual information presented using only two dimensions.
  • Appropriate brightness - As a user with limited vision, I need appropriate brightness for visually presented information for my viewing environment and situation.
  • Appropriate speed - As a user with limited vision or difficulty tracking fast objects, I need to have dynamic images presented at a speed sufficient for me to perceive and track it.
  • Font - As a user with limited vision or other difficulty reading, I need text to be available in a font and style that matches my needs.
  • Magnification - As a user with limited vision, I need to have the magnification of content match my needs given my current viewing situation and environment.
  • Text resizable - As a user with limited vision, I need to be able to resize text to fit my needs in my current viewing situation and environment.
  • Default fonts - As a user with limited vision or other difficulty reading, I need text to be written using a font and style that supports readaibility (example: avoids all caps, italics, and very light weights).
  • Nontext Contrast - As a user with low vision, I need user interface components and graphical objects to have sufficient contrast with adjacent colors.
  • Text alignment - As a user with limited vision or other difficulty reading, I need text not to be set to full justify.
  • Text Contrast - As a user with low vision, I need text to have sufficient contrast with its background.
  • Avoid contrast changes - As a user who has limited vision, I need to avoid when systems might change the foreground color to one that I cannot easily distinguish from the background color.
  • Limited olfactory equivalents - As a user with limited sense of smell or gustation, I need olfactory and gustatory content presented programmatically or I need equivalent visual, auditory, and tactile versions.
  • Customize auditory alternatives - As a user with no or limited hearing, I need to be able to customize the style and speed of alternative content so I can consume it optimally with less distraction from the main content.
  • No vision equivalents - As a user without vision, I need visual content presented programmatically or I need equivalent auditory and tactile versions.
  • Tactile symbol clarity - As someone who feels tactile symbols, I need to have tactile symbols with an appropriate level of clarity so that I can easily perceive, differentiate, or read them.
  • Alternate Versions - As a user with limited ability to interpret content, I need alternative versions of content (iconography, simplified, summarized, etc.)
  • Cultural conventions, words, symbols - As a user with difficulty understanding content provided in unfamiliar or with unfamiliar conventions, I need content and interfaces that use standard conventions, words, and symbols for my culture, with which I am familiar. As a user with a congnitive disability, I must be able to change input and user interface components to use text or symbols with which I am familiar.
  • Number Alternatives - As a user with limited ability to interpret content, I need alternatives to numbers and numeric concepts.
  • Clear Numbers - As a user with limited ability to interpret content, I need familiar, unambiguous numbers.
  • Familiar icons - As a person with limited language skills, I need familiar icons or symbols that help me understand meaning.
  • Focused content - As a person with limited language skills or limited memory, I need relevant content presented in small segments with white space between segments.
  • Plain Language - As a user with limited langauge abilities, I need a version of content that follows plain language principles.
  • scrolling - As a user with limited vision, I need scrolling limited to a single direction.
  • Unexepected movement - As a user with a cognitive and learning disability, I need content and controls to remain in the same relative location and not move unexpectedly.
  • Audio Control - As a user with no vision who uses a screen reader, I must be able to control auditory content presentation (volume, speed, stop/pause)
  • Visual Presentation Control - As a user with limited vision, I must be able to control the visual presentation of content includiong color, orientation, viewport, formatting, spacing, and size.
  • Harmful Sensory Content - As a user with sensory sensitivity, I must be able to identify in advance content that can cause me harm (motion, flashing, haptics?) and control my interaction with it
  • No visual seizure triggers - As a person who may have a reflex seizure to visual stimuli, I need to be able to avoid visual patterns that cause seizures or adjust the viewing of such visual patterns so that they are safe to me.
  • No chemical harm - As a person who has chemical sensitivities, I need to have systems made of safe material and that do not give off chemicals to which I am sensitive or have an allergy.
  • Harmful Triggering Content - As a user with cognitive and mental health sensitivity, I must be able to identify, in advance, sensitive content that may trigger me and control my interaction with it
  • No audio seizure triggers - As a person who may have a reflex seizure to auditory stimuli, I need to to avoid auditory patterns that cause seizures.
  • Non-visual instructions - As a person who lacks the ability to form visual mental images, I need visually oriented instructions to include non-visual cues as well.
  • Visual recognition - As a person who lacks visual memory, I need visual cues that I must recognize, such as icons and logos, to be distinguishable in other ways, such as labels.
  • Adjustable parts - As a user with limited reach, dexterity, or mobility, I need to be able to adjust the location and position of devices and controls or have all of them within easy view and reach.
  • Alternative to holding - As a user with limited reach, dexterity, or mobility, I need an alternative to holding a button down.
  • Control sensitivity - As a user with limited reach, dexterity, or mobility, I need to be able to adjust the sensitivity of controls.
  • Game speed - As a user with limited fine motor control or limited executive functions, I need to be able to adjust the speed of game play.
  • Interface access - As a user with limited reach, dexterity, or mobility, I need to access all areas of the interface with the same input method as the gameplay.
  • Reconfigure controls - As a user with limited reach, dexterity, or mobility, I need to be able to remap and reconfigure my controls.
  • Simple controls - As a user with limited reach, dexterity, or mobility, I need simple controls.
  • Within viewing range - As a user with short stature, mobility issues, or other reasons that I cannot position myself at an expected viewing angle, I need to have information and controls within my viewing range.
  • Large scale layout - As a user of large scale display modes, I visual relationship aspects of the layout to be preserved, and content not to be clipped.
  • AT-compatible - As a user of assistive technology, content and operable elements can be accessed and presented through assistive technology.
  • No vision and no hearing equivalents - As a user with neither vision nor hearing, I need visual and auditory content presented programmatically or I need equivalent tactile versions.
  • Adaptable captions - As a person who uses captions or subtitles to access spoken content, and who also has moderate visual impairment, I need to be able to customize the presentation of captions and subtitles so I can read them effectively.
  • Tactile text speed - As someone who reads tactile text, I need to have tactile text at an appropriate reading speed.
  • Components findable - As a user with a disability, I need to be able to easily find components.
  • Distinguish if actionable - As a user with a disability, I need to be able to distinguish between actionable and non-actionable components, including enabled/disabled controls and different types of components that are/are not actionable.
  • Purposeful labels - As a user with a disability, I need to have labels that make the function of each component be easily perceivable.
  • Recognizable signals - As a user with a disability, I need recognizable signals (including feedback and cues) for different alerts or other messages that use signals.
  • Signposts - As a user with a disability, I need clear signposts to indicate specific locations where functionalities can be found.
  • Separate identification & activation - As someone who has a disability, I need to have separate means of identifying and then activating controls.
  • Turn off visual output - As someone with no vision, I need to have the ability to turn off the visual output from a system.
  • Haptic control - As a user with carpel tunnel or sensitivity to haptics, I need to be able to control haptic output.
  • Personalized Interface - As a user with a disability, I need to be able to customize my interface.

Functional Need: Limited memory & memory loss

There are 25 guidance statements that support this functional need:

  • Consistent Help - As a user with a cognitive disability, I need access to help to be internally consistent
  • Context Sensitive Help - As a user with a cognitive disability, I need context sensitive help.
  • Human Help - As a user with a cognitive disability, I need easy access to trained human help.
  • Error Suggestion - As a user with a cognitive disability, I need error messages to suggest corrections or state that the system is at fault and the user is unable to fix the error.
  • Timeout Warning - As a user with no vision or limited vision, I need to be notified sensorily and programatically when a timeout is about to occur and has occurred.
  • Cognitive Test - As a user with a cognitive disability, I need to complete tasks, including authentication, without cognitive tests.
  • Findable - As a user with a cognitive or learning disability, I need to it to be easy to find what I need.
  • Simple login - As a user with a cognitive and learning disabiltiy, I need a simple way to log in.
  • Preparing for tasks - As a user with a cognitive and learning disabiltiy, I need to know what I will need to complete a task before I begin. .
  • Consistent Identification - As a user with a cognitive disability, I need interactive components with the same functionality to be labeled the same.
  • Distinguishable Controls - Partial 4.2.5 Making Content Usable
  • Clear control behavior - As a user with cognitive and learning disabilites, I need to understand what a control does, how it relates to other content, and how to use it.
  • Consistent Navigation - As a user with a cognitive disability, I need navigation and interactions to be internally consistent
  • Location in Context - As a user with a cognitive disabiilty, I need my current orientation within the larger structure to be presented visually and programmatically
  • Unexepected movement - As a user with a cognitive and learning disability, I need content and controls to remain in the same relative location and not move unexpectedly.
  • Control Distractions - As a user with a cognitive disability, I must be able to control content that can interrupt my focus
  • Redundant Entry - As a user with a cognitive or motor disability, I need to avoid entering data more than once.
  • Non-visual instructions - As a person who lacks the ability to form visual mental images, I need visually oriented instructions to include non-visual cues as well.
  • Visual recognition - As a person who lacks visual memory, I need visual cues that I must recognize, such as icons and logos, to be distinguishable in other ways, such as labels.
  • Focused content - As a person with limited language skills or limited memory, I need relevant content presented in small segments with white space between segments.
  • Follow common interface behavior conventions - As a user with limited or no vision, or difficulty being distracted by and understanding novel layouts and interfaces, I need content and interfaces that are consistent and follow user interface interaction behavior and feedback conventions with which I am familiar.
  • Visual recall - As a person who lacks visual memory, I need time to study visual stimuli that I must later recall, so I can store the relevant information non-visually.
  • Characters and Punctuation - As a user with limited language skills, executive fundtion or memory, I need the characters, symbols, and punctuation required to uderstand ambigous words and phrases.
  • Meaningful organization - As a user with a cognitive and learning disability or an assistive technology user, I need my page and site contents and structure to be organized the site into logical, cohesive sections both sensorily and programmatically.
  • Flexible input format - As a user with limited attention or memory, I need iputs to accept different formats.

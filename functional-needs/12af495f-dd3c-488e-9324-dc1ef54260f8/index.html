Functional Need: No vision

Statements that support this item:

  • Accessible routes - As a user of wayfinding tools, I need the tools to provide a route that is accessible to me, or information about barriers on the way.
  • No vision equivalents - As a user without vision, I need visual content presented programmatically or I need equivalent auditory and tactile versions.
  • No vision interactive equivalent - As a user with no vision, I need interactive components to have meaningful and equivalent sensory and programmatic indicators of their purpose (label), role and value. I need changes in content, state, or value to be programmatically indicated and I need an auditory and tactile indication.
  • Output modality choice - As a user with limited or no vision or with limited or no hearing, I need to be able to choose modalities to be used for outputs from the system.
  • Status indicators - As a person with no or limited vision or with no or limited hearing, I need the status of toggle controls available to me in a modality (e.g., tactile, visual, auditory, etc.) available to me.
  • Limited vision interactive equivalent - As a user with limited vision, I need interactive components to have meaningful and equivalent sensory and programmatic indicators of their purpose (label), role and value. I need changes in content, state, or value to be programmatically indicated or I need a visual, auditory, and tactile indication.
  • Nonvisual Intruction References - As a user wtih limited or no vision, I need instructions that reference visual information to include an additional programmatic reference or provide equivalent auditory or tactile references.
  • Physical layout descriptions - As a user with no vision, I need to have information describing location and layout of operable parts.
  • Error Prevention - As a user with a disability, I need submissions to be checked, confirmable or reversible
  • Language of Content - As a user with no vision or limited language skills, I need the language of content to be programmatically determinable.
  • Meaningful Sequence - As a user with no vision, limited vision, limited fine motor control or no motor control, I need content to be available in a meaningful sequence both sensorily and programmatically.
  • Programmatic structure and relationships - As a user with no vision, limited vision, limited fine motor control or no motor control, I need structure and relationships that are presented sensorily to also be presented programmatically in an equivalent manner (title, labels, section headings).
  • Separate Output Control - As a user of assistive technology, I need to be able to control output of assistive technology separate from output of content.
  • Description of operable parts layout - As a person without vision, I need to have the location and arrangment of operable parts given to me.
  • Meaningful Focus Order - As a user with a mobility disability, I need interactive content to be available in a meaningful sequence that facilitates operation
  • Multiple Paths - As a user with a disability, I need multiple paths through content that provide paths through with unnecessary distractions and interactions.
  • Input modality choice - As a user with a disability, I need to be able to choose modalities to be used for inputs to the system.
  • Follow common layout conventions - As a user with limited or no vision, or difficulty learning or getting an overview of and understanding novel layouts and interfaces,
  • Error Identification - As a user with a sensory disability, I need errors to be identified sensorily and programmatically
  • Follow common interface behavior conventions - As a user with limited or no vision, or difficulty being distracted by and understanding novel layouts and interfaces,
  • Unchanging button functionality - As a user with no vision, limited vision, or a cognitive disability, I need each button or key to have its own functionality rather than functionality that changes depending on context.
  • Unexpected Change of Context - As a user with a disability, I need to be able to input content without unexpected changes in context.
  • Timing - As a user with a disability, I need no time limit OR when I have logged in and a timeout has occurred, I need to be able to continue my task without data loss after logging back in.
  • Timing - As a user with a disability, I have the same amount of control of my personal information/privacy as an individual as other users.
  • Timing - As a user with a disability, I need to be able to adjust, extend or turn off any time out.
  • Nonauditory Intruction References - As a user with no vision, I need the current location of the focus(es) to be presented programmatically.
  • Nonauditory Intruction References - As a user with limited vision, I need a visible indicator of the location of the focus(es) with sufficient contrast.
  • Audio Control - As a user with no vision who uses a screen reader, I must be able to control auditory content presentation (volume, speed, stop/pause)
  • Recognize human - As a person whose appearance is different from the majority of people, I need human-detection tools to recognize me with equal reliability to everyone else, for purpose of obstacle avoidance, non-bot verification, etc. This includes tools operated by other people.
  • Unexpected Change of Focus - As a user with a disability, I need to be able to focus on an element without unexpected changes in context.
  • Recognize individual - As a person whose disability impacts biometric factors such as facial appearance, iris pattern, or finger presence, I need biometric verification tools to provide ways to verify that I am who I say I am with equal reliability to other people.
  • No disruption when activating access feature - As a person who uses access features or modes, I want to be able to activate my desired modality without a serious disruption such as restarting the computer, closing and restarting an app, or leaving an active task in progress.

Functional Need: Limited attention & concentration

Statements that support this item:

  • Error Prevention - As a user with a disability, I need submissions to be checked, confirmable or reversible
  • Consistent Help - As a user with a cognitive disability, I need access to help to be internally consistent
  • Context Sensitive Help - As a user with a cognitive disability, I need context sensitive help.
  • Human Help - As a user with a cognitive disability, I need easy access to trained human help.
  • Speech with visual highlighting - As a user who has difficult reading, I need the individual words highlighted syncronously with the text that is being spoken.
  • Separate Output Control - As a user of assistive technology, I need to be able to control output of assistive technology separate from output of content.
  • Error Identification - As a user with a sensory disability, I need errors to be identified sensorily and programmatically
  • Error Suggestion - As a user with a cognitive disability, I need error messages to suggest corrections or state that the system is at fault and the user is unable to fix the error.
  • Timeout Warning - When timeouts exist, I need to be warned about the timeout in advance and directly before the timeout.
  • Hide Accessibility Features - As a person who can be easily distracted, I need to be able to temporarily hide specific accessibility functions.
  • Unchanging button functionality - As a user with no vision, limited vision, or a cognitive disability, I need each button or key to have its own functionality rather than functionality that changes depending on context.
  • Unexpected Change of Context - As a user with a disability, I need to be able to input content without unexpected changes in context.
  • Timing - As a user with a disability, I need no time limit OR when I have logged in and a timeout has occurred, I need to be able to continue my task without data loss after logging back in.
  • Timeout Warning - As a user with a cognitive disability, I need to complete tasks, including authentication, without cognitive tests.
  • Timing - As a user with a disability, I have the same amount of control of my personal information/privacy as an individual as other users.
  • Timing - As a user with a disability, I need to be able to adjust, extend or turn off any time out.
  • Consistent Identification - As a user with a cognitive disability, I need interactive components with the same functionality to be labeled the same.
  • Distinguishable Controls - As a user with a cognitive disability, I need interactive components to be clearly distinguishable visually and programmatically from static content
  • Location in Context - As a user with a cognitive disability, I need navigation and interactions to be internally consistent
  • Location in Context - As a user with a cognitive disabiilty, I need my current orientation within the larger structure to be presented visually and programmatically
  • Control Distractions - As a user with a cognitive disability, I must be able to control content that can interrupt my focus
  • Multiple Paths - As a user with a disability, I need multiple paths through content that provide paths through with unnecessary distractions and interactions.
  • Redundant Entry - As a user with a cognitive or motor disability, I need to avoid entering data more than once.
  • Unexpected Change of Focus - As a user with a disability, I need to be able to focus on an element without unexpected changes in context.

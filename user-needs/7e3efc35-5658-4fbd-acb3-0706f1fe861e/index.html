User Need: Operate

Statements that support this item:

  • Target Size - As a user with a mobility disability, I need target sizes to be easy to hit
  • Concurrent Input - As a user with a limited or no fine motor control or limited ability to move body, I need to use concurrent input mechanisms
  • Input modality choice - As a user with a disability, I need to be able to choose modalities to be used for inputs to the system.
  • Meaningful Sequence - As a user with no vision, limited vision, limited fine motor control or no motor control, I need content to be available in a meaningful sequence both sensorily and programmatically.
  • Programmatic structure and relationships - As a user with no vision, limited vision, limited fine motor control or no motor control, I need structure and relationships that are presented sensorily to also be presented programmatically in an equivalent manner (title, labels, section headings).
  • Speech variations - As a user with non-typical speech characteristics, such as from voice production difficulty or a Deaf accent, I need voice input systems to recognize my input as well as it does mainstream accents.
  • Description of operable parts layout - As a person without vision, I need to have the location and arrangment of operable parts given to me.
  • Meaningful Focus Order - As a user with a mobility disability, I need interactive content to be available in a meaningful sequence that facilitates operation
  • Unchanging button functionality - As a user with no vision, limited vision, or a cognitive disability, I need each button or key to have its own functionality rather than functionality that changes depending on context.
  • Timing - As a user with a disability, I need no time limit OR when I have logged in and a timeout has occurred, I need to be able to continue my task without data loss after logging back in.
  • Timeout Warning - As a user with a cognitive disability, I need to complete tasks, including authentication, without cognitive tests.
  • Single Action - As a user with a mobility disability, I need keypad style entry to be available for anything beyond a single yes/no based process. The type of keypad supported will vary based on functionality.
  • Single Action - As a user with a mobility disability, I need an option that requires only a single action and a single pointer to input information and navigate (alternative to multipoint gestures, dragging, pointer movements, etc)

User Need: Operate

There are 40 guidance statements that support this user need:

  • Target Size - As a user with a mobility disability, I need target sizes to be easy to hit
  • Concurrent Input - As a user with a limited or no fine motor control or limited ability to move body, I need to use concurrent input mechanisms
  • Input modality choice - As a user with a disability, I need to be able to choose modalities to be used for inputs to the system.
  • Meaningful Sequence - As a user with no vision, limited vision, limited fine motor control or no motor control, I need content to be available in a meaningful sequence both sensorily and programmatically.
  • Programmatic structure and relationships - As a user with no vision, limited vision, limited fine motor control or no motor control, I need structure and relationships (title, labels, section headings) presented sensorily and programmatically.
  • Speech variations - As a user with non-typical speech characteristics, such as from voice production difficulty or a Deaf accent, I need voice input systems to recognize my input as well as it does mainstream accents.
  • Description of operable parts layout - As a person without vision, I need to have the location and arrangment of operable parts given to me.
  • Meaningful Focus Order - As a user with a mobility disability, I need interactive content to be available in a meaningful sequence that facilitates operation
  • Unchanging button functionality - As a user with no vision, limited vision, or a cognitive disability, I need each button or key to have its own functionality rather than functionality that changes depending on context.
  • Flexible input format - As a user with limited attention or memory, I need iputs to accept different formats.
  • Software control - As a user who may have difficulty operating some physical hardware control or peripherals, I need to have the ability to access functionality through software that is ordinarily controlled through hardware.
  • Timeout Warning - As a user with no vision or limited vision, I need to be notified sensorily and programatically when a timeout is about to occur and has occurred.
  • Cognitive Test - As a user with a cognitive disability, I need to complete tasks, including authentication, without cognitive tests.
  • Appropriate challenging - As a user who may have difficulty completing some tasks on my own, I need to have the complexity or difficulty set at a level which is challenging and motivating but not discouraging to me.
  • Emotional Coping - As a user who may have difficulty completing some tasks on my own, I need to to have assistance in coping with my emotional reactions to using the system.
  • Encouragement on task progress - As a user who may have difficulty completing some tasks on my own, I need to receive encouragement that the I am progressing with a task .
  • Goal & Value reminder - As a user who may have difficulty completing some tasks on my own, I need to receive periodic reminders of the goal being worked on and its value.
  • Successes Acknowledged - As a user who may have difficulty completing some tasks on my own, I need to have even small successes acknowledged to increase self-esteem.
  • Simple login - As a user with a cognitive and learning disabiltiy, I need a simple way to log in.
  • Preparing for tasks - As a user with a cognitive and learning disabiltiy, I need to know what I will need to complete a task before I begin. .
  • Information on task progress - As a user who may have difficulty completing some tasks on my own, I need to receive information regarding the progress of completing a task.
  • Sequence of task steps - As a user who may have difficulty completing some tasks on my own, I need to receive prompts identifying the appropriate next step to perform for a task.
  • Findable - As a user with a cognitive or learning disability, I need to it to be easy to find what I need.
  • Not cause fatigue or discomfort - As a person who fatigues easily, I need to have system that do not cause fatigue or discomfort when used.
  • scrolling - As a user with limited vision, I need scrolling limited to a single direction.
  • Unexepected movement - As a user with a cognitive and learning disability, I need content and controls to remain in the same relative location and not move unexpectedly.
  • Keypad Entry - As a user with a mobility disability, I need keypad style entry to be available for anything beyond a single yes/no based process. The type of keypad supported will vary based on functionality.
  • Keyboard shortcuts - As a user who depends on the keyboard or who uses voice control, I need keyboard shortcuts to be predictable and not override existing keyboard shortcuts.
  • Single Action - As a user with a mobility disability, I need an option that requires only a single action and a single pointer to input information and navigate (alternative to multipoint gestures, dragging, pointer movements, etc)
  • Modified motion-based input - As a person who fatigues easily, I need motion-based input tools to accommodate my needs for reduced or slowed motion and rest breaks.
  • Alternatives to motion-based input - As a person who fatigues easily, I need alternatives to motion-based input.
  • Recognize individual - As a person whose disability impacts biometric factors such as facial appearance, iris pattern, or finger presence, I need biometric verification tools to provide ways to verify that I am who I say I am with equal reliability to other people.
  • Timing - As a user with a disability, I need no time limit OR when I have logged in and a timeout has occurred, I need to be able to continue my task without data loss after logging back in.
  • Manage individualization functions - As a user who uses accessibility functions either built into platforms or as part of assitive technologies, I need a way to perform, undo, and redo individualization actions.
  • Manage individualization features - As a user who uses accessibility features either built into platforms or as part of assitive technologies, I need a way to activate or deactivate individualization features.
  • Reset accessibility functions - As a user who uses accessibility functions either built into platforms or as part of assitive technologies, I need a way to reset accessibility functions to their initial states.

  • Meaningful organization - As a user with a cognitive and learning disability or an assistive technology user, I need my page and site contents and structure to be organized the site into logical, cohesive sections both sensorily and programmatically.
  • No Navigation Traps - As a sequential input user, I need to be able to move to and away from any interactive element using any user agent or input modality.
  • User Feedback - As a user with a disability, I need an accessible way to provide user feedback.

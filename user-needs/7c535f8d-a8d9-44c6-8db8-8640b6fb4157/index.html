User Need: Understand / Learn

There are 20 guidance statements that support this user need:

  • Consistent Help - As a user with a cognitive disability, I need access to help to be internally consistent
  • Context Sensitive Help - As a user with a cognitive disability, I need context sensitive help.
  • Human Help - As a user with a cognitive disability, I need easy access to trained human help.
  • Speech with visual highlighting - As a user who has difficult reading, I need the individual words highlighted syncronously with the text that is being spoken.
  • Alternate Versions - As a user with limited ability to interpret content, I need alternative versions of content (iconography, simplified, summarized, etc.)
  • Cultural conventions, words, symbols - As a user with difficulty understanding content provided in unfamiliar or with unfamiliar conventions, I need content and interfaces that use standard conventions, words, and symbols for my culture, with which I am familiar. As a user with a congnitive disability, I must be able to change input and user interface components to use text or symbols with which I am familiar.
  • Error Suggestion - As a user with a cognitive disability, I need error messages to suggest corrections or state that the system is at fault and the user is unable to fix the error.
  • Timeout Warning - When timeouts exist, I need to be warned about the timeout in advance and directly before the timeout.
  • Hide Accessibility Features - As a person who can be easily distracted, I need to be able to temporarily hide specific accessibility functions.
  • Follow common interface behavior conventions - As a user with limited or no vision, or difficulty being distracted by and understanding novel layouts and interfaces,
  • Follow common layout conventions - As a user with limited or no vision, or difficulty learning or getting an overview of and understanding novel layouts and interfaces,
  • Consistent Identification - As a user with a cognitive disability, I need interactive components with the same functionality to be labeled the same.
  • Distinguishable Controls - As a user with a cognitive disability, I need interactive components to be clearly distinguishable visually and programmatically from static content
  • Location in Context - As a user with a cognitive disability, I need navigation and interactions to be internally consistent
  • Location in Context - As a user with a cognitive disabiilty, I need my current orientation within the larger structure to be presented visually and programmatically
  • Identify speaker - As a person who uses captions to understand spoken content, I need to be able to identify easily the relationship between caption text and speaker.
  • Non-visual instructions - As a person who lacks the ability to form visual mental images, I need visually oriented instructions to include non-visual cues as well.
  • Visual recognition - As a person who lacks visual memory, I need visual cues that I must recognize, such as icons and logos, to be distinguishable in other ways, such as labels.
  • Speaker location - As a person using captions in a 3d interface like XR, I need to be able to identify where in space the speaker being captioned is located.
  • Visual recall - As a person who lacks visual memory, I need time to study visual stimuli that I must later recall, so I can store the relevant information non-visually.

Unchanging button functionality

For systems that require physical operation, people may need each button or key to have its own functionality rather than functionality that changes depending on context.

Comment on this item

Notes

For example:

  • Label changes to soft keys can be missed by users
  • Common shortcut keystrokes vary by system mode or context

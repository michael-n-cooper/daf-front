Guidance statements

  • Accessible routes - As a user of wayfinding tools, I need the tools to provide a route that is accessible to me, or information about barriers on the way.
  • Alternate Versions - As a user with limited ability to interpret content, I need alternative versions of content (iconography, simplified, summarized, etc.)
  • Audio Control - As a user with no vision who uses a screen reader, I must be able to control auditory content presentation (volume, speed, stop/pause)
  • Concurrent Input - As a user with a limited or no fine motor control or limited ability to move body, I need to use concurrent input mechanisms
  • Consistent Help - As a user with a cognitive disability, I need access to help to be internally consistent
  • Consistent Identification - As a user with a cognitive disability, I need interactive components with the same functionality to be labeled the same.
  • Context Sensitive Help - As a user with a cognitive disability, I need context sensitive help.
  • Control Distractions - As a user with a cognitive disability, I must be able to control content that can interrupt my focus
  • Cultural conventions, words, symbols - As a user with difficulty understanding content provided in unfamiliar or with unfamiliar conventions, I need content and interfaces that use standard conventions, words, and symbols for my culture, with which I am familiar. As a user with a congnitive disability, I must be able to change input and user interface components to use text or symbols with which I am familiar.
  • Customize auditory alternatives - As a user with no or limited hearing, I need to be able to customize the style and speed of alternative content so I can consume it optimally with less distraction from the main content.
  • Description of operable parts layout - As a person without vision, I need to have the location and arrangment of operable parts given to me.
  • Distinguishable Controls - As a user with a cognitive disability, I need interactive components to be clearly distinguishable visually and programmatically from static content
  • Error Identification - As a user with a sensory disability, I need errors to be identified sensorily and programmatically
  • Error Prevention - As a user with a disability, I need submissions to be checked, confirmable or reversible
  • Error Suggestion - As a user with a cognitive disability, I need error messages to suggest corrections or state that the system is at fault and the user is unable to fix the error.
  • Follow common interface behavior conventions - As a user with limited or no vision, or difficulty being distracted by and understanding novel layouts and interfaces,
  • Follow common layout conventions - As a user with limited or no vision, or difficulty learning or getting an overview of and understanding novel layouts and interfaces,
  • Harmful Sensory Content - As a user with sensory sensitivity, I must be able to identify in advance content that can cause me harm (motion, flashing, haptics?) and control my interaction with it
  • Harmful Triggering Content - As a user with cognitive and mental health sensitivity, I must be able to identify, in advance, sensitive content that may trigger me and control my interaction with it
  • Hide Accessibility Features - As a person who can be easily distracted, I need to be able to temporarily hide specific accessibility functions.
  • Hover Content - As a user with limited vision, I need control content that appears on hover or focus.
  • Human Help - As a user with a cognitive disability, I need easy access to trained human help.
  • Input modality choice - As a user with a disability, I need to be able to choose modalities to be used for inputs to the system.
  • Language of Content - As a user with no vision or limited language skills, I need the language of content to be programmatically determinable.
  • Limited olfactory equivalents - As a user with limited sense of smell or gustation, I need olfactory and gustatory content presented programmatically or I need equivalent visual, auditory, and tactile versions.
  • Limited vision distinguish distractions - As a user with limited vision, I need content that is not related to my task or from a different source than the main content, to be clearly indicated both programmatically, visually with high luminocity, and with equivalent auditory and tactile versions.
  • Limited vision interactive equivalent - As a user with limited vision, I need interactive components to have meaningful and equivalent sensory and programmatic indicators of their purpose (label), role and value. I need changes in content, state, or value to be programmatically indicated and I need a visual, auditory, and tactile indication.
  • Limited vision interactive equivalent - As a user with limited vision, I need interactive components to have meaningful and equivalent sensory and programmatic indicators of their purpose (label), role and value. I need changes in content, state, or value to be programmatically indicated or I need a visual, auditory, and tactile indication.
  • Limited vision locate focus - As a user with limited vision, As a user with limited vision, I need the current location of the focus(es) to be presented programmatically, visually with high contrast/luminosity, and not visually obscured by other content.
  • Location in Context - As a user with a cognitive disability, I need navigation and interactions to be internally consistent
  • Location in Context - As a user with a cognitive disabiilty, I need my current orientation within the larger structure to be presented visually and programmatically
  • Meaningful Focus Order - As a user with a mobility disability, I need interactive content to be available in a meaningful sequence that facilitates operation
  • Meaningful Sequence - As a user with no vision, limited vision, limited fine motor control or no motor control, I need content to be available in a meaningful sequence both sensorily and programmatically.
  • Motion Errors - As a user with a mobility disability, I need to turn off movement based input.
  • Multiple Paths - As a user with a disability, I need multiple paths through content that provide paths through with unnecessary distractions and interactions.
  • No hearing equivalents - As a user without hearing or with limited hearing, I need auditory content presented programmatically or I need equivalent visual and tactile versions.
  • No vision and no hearing equivalents - As a user with neither vision nor hearing, I need visual and auditory content presented programmatically or I need equivalent tactile versions.
  • No vision equivalents - As a user without vision, I need visual content presented programmatically or I need equivalent auditory and tactile versions.
  • No vision interactive equivalent - As a user with no vision, I need interactive components to have meaningful and equivalent sensory and programmatic indicators of their purpose (label), role and value. I need changes in content, state, or value to be programmatically indicated and I need an auditory and tactile indication.
  • No-color equivalents - As a user with low vision or color vision deficiency, I need information provided through chromaticity to be available in another visual manner.
  • Nonauditory Instruction References - As a user with a limited or no hearing, I need instructions that reference auditory information to include an additional programmatic reference or provide equivalent visual or tactile references
  • Nonauditory Intruction References - As a user with no vision, I need the current location of the focus(es) to be presented programmatically.
  • Nonauditory Intruction References - As a user with limited vision, I need a visible indicator of the location of the focus(es) with sufficient contrast.
  • Nongustatory Alternate Intruction References - As a user with a limited sense of smell and gustation, I need instructions that reference smell or taste to include an additional programmatic reference or provide equivalent visual, auditory, and tactile references
  • Nontactile Alternate Intruction References - As a user with a limited tactile sensitivity, I need instructions that reference touch to include an additional programmatic reference or provide equivalent visual or auditory references
  • Nonvisual Intruction References - As a user wtih limited or no vision, I need instructions that reference visual information to include an additional programmatic reference or provide equivalent auditory or tactile references.
  • Nonvisual and Auditory Alternate Intruction References - As a user with no vision and no hearing, I need instructions that reference auditory or visual information to include an additional programmatic reference or provide equivalent tactile references
  • Noticeable non-auditory alert - As a user with no hearing, I need perceivable forms of time-sensitive alerts to be in a format that I will notice immediately, such as flashing lights or insistent vibration.
  • Notifications in auditory alternatives - As a user with low or no hearing, I need notifications to work with visual or tactile equivalents for auditory content.
  • Output modality choice - As a user with limited or no vision or with limited or no hearing, I need to be able to choose modalities to be used for outputs from the system.
  • Perceive auditory content - As a user with low or no hearing, I need visual or tactile equivalents for auditory content.
  • Physical layout descriptions - As a user with no vision, I need to have information describing location and layout of operable parts.
  • Pointer Errors - As a user with a motor impairment, I need to be able to correct or under pointer mistakes
  • Programmatic structure and relationships - As a user with no vision, limited vision, limited fine motor control or no motor control, I need structure and relationships that are presented sensorily to also be presented programmatically in an equivalent manner (title, labels, section headings).
  • Redundant Entry - As a user with a cognitive or motor disability, I need to avoid entering data more than once.
  • Separate Output Control - As a user of assistive technology, I need to be able to control output of assistive technology separate from output of content.
  • Single Action - As a user with a mobility disability, I need keypad style entry to be available for anything beyond a single yes/no based process. The type of keypad supported will vary based on functionality.
  • Single Action - As a user with a mobility disability, I need an option that requires only a single action and a single pointer to input information and navigate (alternative to multipoint gestures, dragging, pointer movements, etc)
  • Speech variations - As a user with non-typical speech characteristics, such as from voice production difficulty or a Deaf accent, I need voice input systems to recognize my input as well as it does mainstream accents.
  • Speech with visual highlighting - As a user who has difficult reading, I need the individual words highlighted syncronously with the text that is being spoken.
  • Status indicators - As a person with no or limited vision or with no or limited hearing, I need the status of toggle controls available to me in a modality (e.g., tactile, visual, auditory, etc.) available to me.
  • Supplement auditory location cues - As a user with no hearing, I need auditory cues to location in space to be represented in other forms, such as visual, textual, or haptic cues.
  • Synchronized auditory alternatives - As a user with limited hearing or limited language skills, I need alternate content, such as subtitles, to be synchronized with the main content.
  • Target Size - As a user with a mobility disability, I need target sizes to be easy to hit
  • Timeout Warning - When timeouts exist, I need to be warned about the timeout in advance and directly before the timeout.
  • Timeout Warning - As a user with a cognitive disability, I need to complete tasks, including authentication, without cognitive tests.
  • Timing - As a user with a disability, I need no time limit OR when I have logged in and a timeout has occurred, I need to be able to continue my task without data loss after logging back in.
  • Timing - As a user with a disability, I have the same amount of control of my personal information/privacy as an individual as other users.
  • Timing - As a user with a disability, I need to be able to adjust, extend or turn off any time out.
  • Unchanging button functionality - As a user with no vision, limited vision, or a cognitive disability, I need each button or key to have its own functionality rather than functionality that changes depending on context.
  • Unexpected Change of Context - As a user with a disability, I need to be able to input content without unexpected changes in context.
  • Unexpected Change of Focus - As a user with a disability, I need to be able to focus on an element without unexpected changes in context.
  • Visual Presentation Control - As a user with limited vision, I must be able to control the visual presentation of content includiong color, orientation, viewport, formatting, spacing, and size.
  • Voice Input Errors - As a user with a mobility disability who uses voice control, I need to minimize mistakes that could occur because of mistaken speech (e.g. operations are tied to a single letter shortcut)
